# DOTween Basics

Sub-doc of [dotween-design](./SKILL.md). Covers what happens between "import the package" and "write your first tween".

## `DOTween.Init` — the entry point

Two overloads (`DOTween.cs:227,243`):

```csharp
public static IDOTweenInit Init(bool? recycleAllByDefault = null,
                                 bool? useSafeMode = null,
                                 LogBehaviour? logBehaviour = null);

// internal full init (auto-called)
static IDOTweenInit Init(DOTweenSettings settings, bool? recycleAllByDefault,
                          bool? useSafeMode, LogBehaviour? logBehaviour);
```

Returns `IDOTweenInit` so you can chain:

```csharp
DOTween.Init(recycleAllByDefault: true, useSafeMode: true, logBehaviour: LogBehaviour.Default)
    .SetCapacity(tweenersCapacity: 500, sequencesCapacity: 50);
```

### When to call Init explicitly

- Before the first tween on performance-critical scenes where capacity must be pre-sized.
- To read `DOTweenSettings` at a known point for debugging.
- To override Safe Mode / log behaviour in automated test runs.

Otherwise: **don't call it**. `AutoInit` at `DOTween.cs:236` runs on first tween creation and loads the `DOTweenSettings` Resource.

### Idempotency

```csharp
if (initialized) return instance; // DOTween.cs:229
```

Safe to call multiple times. Second call is a no-op even if parameters differ — re-init is NOT supported. Use `DOTween.Clear(destroy: false)` to reset state without destroying the driver.

## The `[DOTween]` driver GameObject

`DOTween.cs:254` calls `DOTweenComponent.Create()`:

```csharp
// DOTweenComponent.cs:268-270
GameObject go = new GameObject("[DOTween]");
DontDestroyOnLoad(go);
DOTween.instance = go.AddComponent<DOTweenComponent>();
```

Properties of the driver:
- Name `[DOTween]` (square brackets — visible in hierarchy during Play).
- `DontDestroyOnLoad` — survives scene transitions.
- On duplicate instances (rare — happens if `DOTween.cs` is stripped differently across asmdefs), the duplicate logs a warning and self-destroys (`DOTweenComponent.cs:41`).

The driver's `Update / LateUpdate / FixedUpdate` advance every active tween. Its `OnApplicationQuit` sets `DOTween.isQuitting = true` to prevent post-quit tween creation.

**Do not destroy `[DOTween]` manually.** Use `DOTween.Clear(destroy: true)` if you need to tear it down (Test Runner scenarios).

## `DOTweenSettings` Resource

`Resources.Load(DOTweenSettings.AssetName)` at `DOTween.cs:232`. The asset lives at `Assets/Resources/DOTweenSettings.asset` after running the DOTween Utility Panel "Open Settings" command.

Settings copied into statics on Init (`DOTween.cs:257-277`):
- `useSafeMode`, `logBehaviour`
- `defaultRecyclable`, `defaultAutoPlay`, `defaultUpdateType`, `defaultTimeScaleIndependent`
- `defaultEaseType`, `defaultEaseOvershootOrAmplitude`, `defaultEasePeriod`
- `timeScale`, `unscaledTimeScale`
- `useSmoothDeltaTime`, `maxSmoothUnscaledTime`, `rewindCallbackMode`
- `safeModeOptions.logBehaviour`, `safeModeOptions.nestedTweenFailureBehaviour`

Changing the asset at runtime does NOT take effect — only fields mutated directly on `DOTween.*` statics apply mid-game.

## Modules — the DOTweenModule*.cs generators

DOTween 1.2+ splits per-package extensions into **Modules**: separate `.cs` files generated by the Utility Panel ("Create ASMDEF" or "Add/Remove Modules"). In source form they live at `_DOTween.Assembly/bin/Modules/`:

| Module file | Exposes | Triggered by |
|-------------|---------|--------------|
| `DOTweenModuleAudio.cs` | `AudioSource.DOFade/DOPitch`, `AudioMixer.DOSetFloat` + control methods | `UnityEngine.AudioModule` |
| `DOTweenModulePhysics.cs` | `Rigidbody.DOMove/DORotate/DOLookAt/DOJump` | `UnityEngine.PhysicsModule` |
| `DOTweenModulePhysics2D.cs` | `Rigidbody2D.DOMove/DORotate/DOJump` | `UnityEngine.Physics2DModule` |
| `DOTweenModuleSprite.cs` | `SpriteRenderer.DOColor/DOFade` | `UnityEngine.SpriteAnimationModule` |
| `DOTweenModuleUI.cs` | `RectTransform.DOAnchorPos...`, `CanvasGroup.DOFade`, `Image/Text.DO*` | uGUI (`com.unity.ugui`) |
| `DOTweenModuleUIToolkit.cs` | VisualElement tweens (experimental) | UI Toolkit |
| `DOTweenModuleUnityVersion.cs` | **Async APIs** (`AsyncWaitForCompletion/Kill/Rewind/ElapsedLoops/Position`), `WaitForStart`, version-guarded helpers | Unity version-specific |
| `DOTweenModuleEPOOutline.cs` | Third-party EPO Outline integration | Only if EPO Outline package present |
| `DOTweenModuleUtils.cs` | Editor utility / DOTween setup helpers | Internal utility |

**`AsyncWaitFor*` APIs live in `DOTweenModuleUnityVersion.cs:216+`** gated by `#if UNITY_2018_1_OR_NEWER && (NET_4_6 || NET_STANDARD_2_0)`. If this file is not generated (checkbox unchecked in Utility Panel), the async methods don't compile.

### Symptoms of missing Modules

- `CS1061: Tween does not contain a definition for 'AsyncWaitForCompletion'` → UnityVersion module missing
- `CS1061: Rigidbody does not contain a definition for 'DOMove'` → Physics module missing
- `CS1061: Image does not contain a definition for 'DOFade'` → UI module missing

Fix: Tools → Demigiant → DOTween Utility Panel → Setup DOTween → toggle the module, click Apply.

## `useSafeMode` — what it actually does

`DOTween.cs:49,51`:

```csharp
public static bool useSafeMode = true;
public static SafeModeLogBehaviour safeModeLogBehaviour = SafeModeLogBehaviour.Warning;
```

Safe Mode wraps the per-tween `DoUpdate` in a try/catch. When a missing target / null plugin / state corruption is caught:

1. The tween is killed internally.
2. Depending on `safeModeLogBehaviour`:
   - `None` → silent
   - `Warning` → `Debug.LogWarning`
   - `Error` → `Debug.LogError`
3. `nestedTweenFailureBehaviour`:
   - `TryToPreserveSequence` (default) — parent sequence keeps running
   - `KillWholeSequence` — parent sequence dies

**When to disable**: during development, set `useSafeMode = false` temporarily to get raw NullReferenceException stack traces. For production, keep it ON — the performance cost is small.

## `DOTween.SetTweensCapacity`

`DOTween.cs:301`:

```csharp
public static void SetTweensCapacity(int tweenersCapacity, int sequencesCapacity);
```

Defaults: 200 Tweeners + 50 Sequences. When exceeded, DOTween logs a warning and dynamically expands the pool — one-time allocation but noticeable on frame budget for large bursts.

**Guidance**: Set once at start. For a 2D particle game with hundreds of simultaneous tweens, 1000 / 100 is reasonable. For a turn-based UI game, defaults are fine.

## `DOTween.Clear`

`DOTween.cs:311`:

```csharp
public static void Clear(bool destroy = false);
```

- `destroy: false` → kills all tweens and resets state, but keeps the `[DOTween]` GameObject. Next tween creation reuses it.
- `destroy: true` → also destroys `[DOTween]`. Use in Test Runner teardown to fully reset between tests.

Do NOT call `Clear(destroy: true)` mid-game unless you're sure no pending tween references outlive the call.

## `DOTween.timeScale` vs `Time.timeScale`

`DOTween.cs:65-69`:

```csharp
public static float timeScale = 1;          // Applied to ALL tweens
public static float unscaledTimeScale = 1;  // Applied only to SetUpdate(UpdateType.X, isIndependentUpdate: true) tweens
```

Effective delta:
- Normal tween: `Unity.Time.timeScale * DOTween.timeScale * tween.timeScale`
- Independent tween: `DOTween.unscaledTimeScale * DOTween.timeScale * tween.timeScale`

Common pattern for slow-motion or game-pause:
```csharp
Time.timeScale = 0;              // Pauses gameplay
DOTween.timeScale = 1;           // DOTween animations still run IF they opt in via SetUpdate(X, true)
```

## Basics checklist

- [ ] Explicit `DOTween.Init` only when capacity / Safe Mode / log must be controlled.
- [ ] `[DOTween]` GameObject never destroyed manually.
- [ ] Required Modules generated via Utility Panel (UnityVersion for async, Physics / UI as needed).
- [ ] `DOTweenSettings` Resource committed to the project (not in `.gitignore`).
- [ ] `useSafeMode` left ON in release builds; temporarily OFF during bug hunts.
- [ ] `SetTweensCapacity` tuned for the scene with the highest concurrent tween count.
- [ ] `DOTween.Clear(destroy: true)` in Test Runner teardown only.
