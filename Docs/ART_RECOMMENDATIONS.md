# 大学生模拟器美术改进建议报告

日期：2026-06-28

## 1. 当前项目判断

项目已经从旧 Kings 资源包迁移到 `Assets/UniversitySimulator` 目录，主场景是 `Assets/UniversitySimulator/Scenes/Game.unity`，UI 使用 UGUI。当前画面已经有大学主题的基础替换：米色背景、四项状态图标、中文卡牌内容、部分事件卡图。

但整体视觉仍然混着旧 Kings 风格。最明显的是：卡牌仍是金黄色和王冠符号，菜单和面板还大量引用旧 `LegacyKingsUI` 资源，字体仍主要是 `SourceSansPro-Regular.ttf`，部分成就/日志/旧功能文案还有 king、kingdom 等旧主题痕迹。

这不是大问题，反而说明现在最适合做“低成本统一风格”：先换少数共享资源和颜色，不要逐个重做界面。

## 2. 建议的整体方向

建议把美术方向定为“校园手账 + 轻量决策卡牌”。

核心目标：

- 背景像大学生活环境或纸面手账，而不是空白舞台。
- 卡牌像课程表、校园通知、社团传单或学生证，而不是王国卡。
- UI 像手机/校园 App 的信息条，而不是中世纪资源包菜单。
- 图标统一成清晰的线面结合图标，避免太细、太白、面积太小。

推荐色盘：

| 用途 | 建议颜色 | 用法 |
| --- | --- | --- |
| 主文字 | `#1F2936` 深墨色 | 卡牌标题、正文、菜单文字 |
| 主界面底色 | `#F2E7D2` 暖纸色 | 背景或浅面板 |
| 主强调色 | `#2F6F73` 校园青绿 | 卡背、按钮、标题条 |
| 学业色 | `#3A6EA5` 蓝 | 学业图标、学习类卡牌强调 |
| 人际色 | `#B85C8E` 玫紫 | 社交、人际类反馈 |
| 经济色 | `#C9972B` 琥珀 | 经济类反馈 |
| 身心色 | `#D96A5E` 珊瑚 | 健康、压力、疲惫 |

重点是不要继续让背景、卡牌、图标都偏黄。现在金黄卡牌和米色背景放在一起，容易显得单调，也会削弱大学主题。

## 3. 第一优先级：一处改动影响最大的事项

### 3.1 统一卡牌样式

当前 63 张卡全部使用 `cs_None`：

- 普通卡：46 张
- 主线卡：16 张
- 真结局卡：1 张

`cs_None` 现在仍是金黄色卡色，并引用旧王冠图标 `crown_lge.png`。这是“看起来仍像 Kings”的最大来源。

建议做法：

1. 新建或复制一个 `cs_University` 卡牌样式。
2. 把 `cardColor` 从金黄改为校园青绿或浅纸色。
3. 把中心默认图标从王冠换成大学主题图标，例如书本、学生证、课程表、校徽抽象图形。
4. 准备一张新的卡背：`card_back_university.png`，尺寸沿用 `840 x 1240`。
5. 把普通卡先统一使用这个样式，再按需要拆出考试周、经济压力、真实自我等子样式。

低成本版本：

- 直接把 `cs_None` 的颜色改为 `#2F6F73` 或 `#F7F1E3`。
- 把图标从王冠换成 `study.png` 或新的校园图标。
- 把卡牌文字改为深墨色，白字只保留在深色条上。

### 3.2 换卡背

旧卡背是灰白王冠风格，主题不匹配。建议做一张很简单的卡背即可：

- 底色：校园青绿或深墨色。
- 图案：淡线条网格、课表格、学生证轮廓、校徽抽象图形。
- 中心不要放王冠。
- 边框保留圆角，方便沿用旧卡牌模板。

这张图能覆盖所有抽牌前/翻牌相关视觉，投入很小，收益很明显。

### 3.3 修正卡图可读性

当前已有 47 张卡图文件，其中导入报告显示：

- 预期卡图：46 张
- 已绑定：44 张
- 缺失：E008、E012
- 多余：E036、E053
- 低可见度明显的卡图：E027、E059、E062、E064、E068、E074、E084

建议美术先处理这 9 项：

| 类型 | 卡牌 | 建议 |
| --- | --- | --- |
| 缺图 | E008 校园比赛 | 补 `Assets/UniversitySimulator/Art/cards/card-E008.png` |
| 缺图 | E012 课堂展示 | 补 `Assets/UniversitySimulator/Art/cards/card-E012.png` |
| 太淡/太小 | E027、E059、E062、E064、E068、E074、E084 | 加粗线条、扩大主体、改深色或加底形 |

卡图规范建议：

- 统一正方形透明 PNG，建议 `1024 x 1024`。
- 图标主体占画布 35% 到 55%。
- 线条不要低于 10 px。
- 不要只用纯白线条；建议用 `#1F2936` 深墨色，或深墨色加一个状态色。
- 如果必须放在深色卡面上，再用白色图标。

## 4. 游玩界面建议

### 4.1 顶部状态栏

当前顶部状态栏已经换成了身心、学业、人际、经济四项，这是正确方向。但顶栏仍像旧资源包的深灰横条，图标和日期信息有点孤立。

建议：

- 顶栏底色改为深墨色 `#1F2936`，透明度 85% 左右；或改为浅纸色面板配深色图标。
- 四个状态图标保留现在的大学主题，但分别上色：身心珊瑚、学业蓝、人际玫紫、经济琥珀。
- 日期区从“第23天”升级为更有校园感的文本，例如“第 23 天 / 周三”或“第 4 周”。
- 如果旧的左侧详细属性条还会在正常游玩中出现，建议默认隐藏。它仍显示 Authority、Intelligence 等旧概念，会和顶部四项冲突。

### 4.2 背景

当前 `background.png` 是暖纸纹理，干净但过空。建议低成本增加 2 到 3 张背景：

- `bg_dorm_night.png`：宿舍夜晚，适合日常/疲惫事件。
- `bg_library_day.png`：图书馆或自习室，适合学习/考试事件。
- `bg_campus_evening.png`：校园傍晚，适合社交/主线事件。

不需要高精度场景图。可以用模糊插画、低饱和照片、手账纸面拼贴。关键是让第一眼知道这是大学生活。

如果暂时只做一张背景，建议在现有米色纸纹上叠很淡的校园元素：课表线、便签、书页边角、校园地图线条。透明度控制在 8% 到 15%，不要抢卡牌。

### 4.3 右侧菜单和底部槽位

当前侧边菜单和底部圆点槽位仍像旧功能占位。建议先做视觉替换，不改逻辑：

- 菜单按钮：三横线可以保留，但背景改成浅色圆角小按钮。
- 空槽位：从黑色圆点改成低透明度虚线方格或小背包格。
- 已有物品图标如果仍是魔法豆、宝箱、王冠类，先替换为校园物件：校园卡、笔记本、耳机、咖啡、雨伞、学生证。
- 暂时不用的按钮可以隐藏，不要留一排空黑点。

## 5. 菜单和面板建议

场景里 `panel.png`、`button.png`、`slider.png`、`SourceSansPro-Regular.ttf` 引用很多，所以菜单最适合做统一换皮。

建议：

1. 把面板底图统一成浅纸色或白色半透明面板。
2. 按钮从旧灰色改成校园青绿描边按钮。
3. 设置、任务、成就、排行榜图标统一重画为同一套线面图标。
4. 暂时保留功能结构，不要先大改布局。
5. 所有还出现 king、kingdom、ruler、crown 的菜单文案，统一改成大学主题。

菜单命名建议：

| 旧功能感 | 大学主题替代 |
| --- | --- |
| Achievements | 回忆 / 成就册 |
| Quests | 待办 / 主线 |
| Inventory | 背包 / 物品 |
| Leaderboard | 记录 / 档案 |
| Settings | 设置 |

## 6. 字体和中文可读性

当前 `SourceSansPro-Regular.ttf` 在场景里有大量引用，不适合作为中文主字体。建议导入一个中文字体：

已选定字体：`Noto Sans CJK SC`

项目内路径：

- 字体文件：`Assets/UniversitySimulator/Art/Fonts/NotoSansCJKsc-Regular.otf`
- 授权文件：`Assets/UniversitySimulator/Art/Fonts/NotoSansCJK_LICENSE.txt`

选择原因：

- 简体中文覆盖完整，适合当前中文卡牌和 UI。
- 字形中性、清晰，不会抢卡牌图标和背景风格。
- 授权是 SIL Open Font License 1.1，适合随项目分发。
- Unity 已能识别为 `Noto Sans CJK SC` 字体资源。

低成本做法：

- 先只替换主场景和卡牌模板里的 Text 字体。
- 标题用 SemiBold 或 Bold。
- 正文用 Regular。
- 卡牌标题建议 32 到 40，正文 22 到 28，选项文字 24 到 30，按实际分辨率微调。

不要让卡牌正文继续用白色小字压在亮黄色卡面上。推荐浅卡深字，或深卡白字，二选一。

## 7. 卡牌分类美术建议

当前所有卡都用 `cs_None`，但 CSV 里的 tags 已经有分类，可以低成本拆出 4 到 5 个卡牌样式。

建议先做这些样式：

| 样式名 | 对应内容 | 视觉 |
| --- | --- | --- |
| `cs_Study` | 课程、考试、学业 | 蓝色标题条、书本/试卷图标 |
| `cs_Social` | 社交、人际、宿舍 | 玫紫标题条、对话/人像图标 |
| `cs_Economy` | 经济、兼职、金钱 | 琥珀标题条、校园卡/钱包图标 |
| `cs_BodyMind` | 身心、健康、生病 | 珊瑚标题条、心率/睡眠图标 |
| `cs_Mainline` | 主线、真实自我 | 深墨或青绿，图标更特殊 |

这一步不用给每张卡做不同卡面，只要按类别改变标题条、中心图标和边框色，就能明显提升信息识别。

## 8. 给美术的最小资源清单

| 优先级 | 资源 | 数量 | 规格建议 |

## 9. 推荐执行顺序

1. 先改 `cs_None` 或新增 `cs_University`，替换卡色、王冠图标、卡背。
2. 补 E008、E012 两张缺失卡图。
3. 修 E027、E059、E062、E064、E068、E074、E084 的可见度。
4. 统一顶部四项状态图标颜色。
5. 替换主菜单、设置、任务、背包等按钮图标。
6. 导入中文字体并替换卡牌模板和主场景关键文本。
7. 再考虑做多张背景和分类卡牌样式。

## 10. 不建议现在做的事

- 不建议逐张手改 63 个卡牌 prefab。后续重新导入卡牌时容易被覆盖。
- 不建议现在删除 `LegacyKingsUI`。场景还有大量引用，先替换再清理。
- 不建议先做复杂动态背景。当前收益最高的是卡牌样式、卡背、图标、字体。
- 不建议继续用大面积金黄色做主视觉。它和旧 Kings 绑定太强。
- 不建议让背景、卡牌、菜单都使用同一种米色。需要用深墨、青绿、蓝、玫紫、琥珀把层级拉开。

## 11. 最快能看到效果的方案

如果只给美术半天时间，建议做：

1. 一张校园卡背。
2. 一个默认校园图标，替换王冠。
3. 把 `cs_None` 卡色改成校园青绿或浅纸色。
4. 补 E008、E012。
5. 把 7 张太淡的卡图改深、加粗、放大。

这几个动作不会改玩法，也不需要新增系统，但能让游戏从“Kings 换了文本”明显变成“大学生模拟器”。
